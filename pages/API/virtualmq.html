VirtualMQ is a HTTP server that allows the communication between PrivateSky nodes and/or agents inside a node, allows the storage and retrieval of bricks (blocks of data stored on the server in files identified by the block's cryptographic hash) and allows the grouping of different bricks under the same unique identifier (anchor). Each one of the three components is explained in more detail at the links below. The VirtualMQ server can also be expanded to include custom services (besides the default ones below). The section How to expand a VirtualMQ server explains in more detail how a VirtualMQ server can be expanded. Brick Storage Anchoring Message Passing The createVirtualMQ() method creates and starts a VirtualMQ server. createVirtualMQ(port, rootFolder, sslConfig, callback) port The port on on which the server is listening for requests.

rootFolder The folder where the server keeps the files uploaded on it.

sslConfig |Optional An object that configures, among others, the communication protocol used to communicate with clients. By default, "http" is used.

callback The function called when the server is started.

 The server instance. The createVirtualMQ() method starts a VirtualMQ server and returns its instance. In the following example, a server that listens on port 8080 and saves files in folder "/foo/tmp" is created. const VirtualMQ = require("virtualmq"); const port = 8080; const server VirtualMQ.createVirtualMQ(port, "/foo/tmp", undefined, (err) => { if (err) { throw err; } console.log("The server was started and is listening on port", port); }); The createVirtualMQ() method starts a HTTP server and return the server instance. As a result, the server can be dynamically expanded using the API below. server.put(url, handler); url The url where the server listens for PUT requests. handler The function called when the server receives a PUT request at the specified url. server.post(url, handler); url The url where the server listens for POST requests. handler The function called when the server receives a POST request at the specified url. server.get(url, handler); url The url where the server listens for GET requests. handler The function called when the server receives a GET request at the specified url.
